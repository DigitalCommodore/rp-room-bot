# RP Room Builder — Setup Guide

A Discord bot with a modern web UI for creating and managing roleplay room descriptions and images.

---

## Prerequisites

- **Node.js** v18 or newer — [download here](https://nodejs.org/)
- A **Discord Bot Token** — see "Creating Your Bot" below

---

## Quick Start

1. Open a terminal in the `rp-room-bot` folder

2. Install dependencies (only needed once):
   ```
   npm install
   ```

3. Build the web UI (only needed once, or after code changes):
   ```
   npm run build
   ```

4. Start the bot + UI:
   ```
   npm start
   ```

5. Open **http://localhost:3000** in your browser

---

## Creating Your Discord Bot

1. Go to the [Discord Developer Portal](https://discord.com/developers/applications)
2. Click **"New Application"** and give it a name
3. Go to the **Bot** tab on the left
4. Click **"Reset Token"** and copy the token — you'll paste this into the app
5. Under **Privileged Gateway Intents**, enable:
   - **Message Content Intent** (required for text operations)
6. Go to **OAuth2 > URL Generator**
7. Under **Scopes**, check: `bot`
8. Under **Bot Permissions**, check:
   - Send Messages
   - Manage Messages (required for pinning)
   - Attach Files
   - Read Message History
9. Copy the generated URL and open it in your browser to invite the bot to your server

## How It Works

### Creating a Room
- Click **"+ Create"** in the sidebar to create a new room config
- Select your server and channel from the dropdowns (or paste IDs manually)
- Write a description — choose between plain text or a rich Discord embed
- Upload images by dragging & dropping or clicking the upload area
- Reorder images with the arrow buttons on hover

### Deploying
- Click **"Deploy to Discord"** to send everything to the channel
- The bot sends the text message first, then each image as a separate message
- Messages are pinned in reverse order so the text appears at the top of the pins panel

### Updating
- Edit the room description, images, or embed settings
- Click **"Update Room"** — the bot edits the existing messages in place
- No messages are deleted or re-created; pins stay intact

### Configurations
- All room configs are saved as JSON files in the `configs/` folder
- Uploaded images are stored in the `uploads/` folder
- You can back these up or share them between machines

## Development Mode

To run with hot-reload for the frontend:
```
npm run dev
```
This starts the Express backend on port 3000 and the Vite dev server on port 5173 (with API proxy to 3000).
